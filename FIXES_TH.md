# สรุปการแก้ไข (TeYoMaRu OS v0.2 → v0.2.1 fixed)

## บั๊กหลักที่ทำให้ใช้งานไม่ได้ (แก้แล้ว)

### 1) จอไม่ขึ้น / สีเพี้ยน / ขึ้นแค่บางส่วนของจอ
**สาเหตุ:** `main.cpp` เรียก `ui_init()` จาก `src/core/ui.cpp` ซึ่งเป็นโค้ดตั้งค่าจอรุ่นเก่าที่ค้างอยู่ในโปรเจกต์
โค้ดตัวนี้ตั้งความละเอียดผิดเป็น 320x240 (จอจริงคือ 480x320 หลังหมุนแนวนอน) และ**ไม่ได้เรียก
`tft.invertDisplay(true)`** ตามที่ README บอกไว้ว่าจำเป็น ทำให้สีดำ/ขาวสลับกัน

ในขณะเดียวกัน `src/core/display.cpp` มีโค้ด `display_init()` ที่ตั้งค่าถูกต้องครบถ้วนอยู่แล้ว
(480x320, invertDisplay, buffer ขนาดเหมาะสม) แต่**ไม่เคยถูกเรียกใช้จากที่ไหนเลย**

**วิธีแก้:** เขียน `ui.cpp` ใหม่ให้เรียก `display_init()` จริง ๆ แทนการตั้งค่าจอซ้ำเอง

### 2) แถบโหลดตอน Boot ค้างที่ 0% ตลอด ไม่เข้า Desktop สักที
**สาเหตุ:** `lv_conf.h` ตั้ง `LV_TICK_CUSTOM 0` ซึ่งหมายความว่าโปรแกรมต้องป้อนเวลาให้ LVGL เอง
ผ่านฟังก์ชัน `lv_tick_inc()` แต่ในโค้ดทั้งโปรเจกต์ไม่มีการเรียกฟังก์ชันนี้เลยสักที่
ทำให้ LVGL ไม่รู้ว่าเวลาผ่านไป และ timer ของแถบโหลด (ที่ต้องขยับทุก 80ms) จึงไม่เคยทำงาน

**วิธีแก้:** เพิ่มการเรียก `lv_tick_inc()` ใน `ui_loop()` โดยคำนวณเวลาที่ผ่านไปจาก `millis()`

## จุดที่ปรับให้สมบูรณ์ขึ้น

### 3) เลย์เอาต์ถูกออกแบบมาสำหรับจอกว้าง 320px แต่จอจริงกว้าง 480px
- `statusbar.cpp`: เปลี่ยนความกว้างจาก fix 320px เป็น `LV_PCT(100)` ให้เต็มจอเสมอ
- `desktop.cpp`: ปรับตำแหน่ง tile ทั้ง 8 ปุ่มให้อยู่กึ่งกลางแนวนอนของจอ 480px (เดิมชิดซ้าย เหลือพื้นที่ว่างขวาจอเยอะ)

### 4) หน่วยความจำ LVGL
- `lv_conf.h`: ปรับ `LV_MEM_SIZE` จาก 32KB เป็น 48KB ให้ตรงกับที่ README ระบุไว้ และเผื่อพื้นที่สำหรับ widget ที่เพิ่มเข้ามา

## ไฟล์ที่มีอยู่แต่ไม่ได้ถูกใช้งาน (ไม่กระทบการทำงาน แต่ควรรู้ไว้)
- `include/User_Setup.h` — ไม่ได้ถูก include เลย เพราะ `platformio.ini` ใช้ `-D USER_SETUP_LOADED`
  ทำให้ TFT_eSPI ไปอ่านค่าจาก build_flags ใน platformio.ini โดยตรงแทน (เก็บไว้เป็นเอกสารอ้างอิงได้)
- `src/widgets/widgets.cpp/.h` และ `src/widgets/popup.cpp/.h` — เป็นฟังก์ชันสำรอง (panel/tile แบบอื่น,
  popup แจ้งเตือน) ที่เขียนไว้แต่ยังไม่ถูกเรียกใช้จากหน้าไหนเลย ใช้เป็นฐานสำหรับต่อยอดฟีเจอร์ในอนาคตได้
